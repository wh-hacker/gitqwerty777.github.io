gdb入門

簡介

gdb是FSF下的一個子計畫，目的是提供一個除錯器的實作。只要使用GNU計畫出產的編譯器，就可以用它來進行除錯。
gdb可以debug C/C++/Objective-C/Fortran/Java 等語言。

基本知識

function -> frame

  • 正在執行的函數就是 frame 0，而呼叫該函數的就是 frame 1，以此類推
  • 在gdb進入另一個frame之前，它會將該frame的一些變數值之類的儲存至堆疊(stack)裡，等到從frame回來後再從這些stack裡把這些變數值取回來

除錯程序

  1. 載入程式
  2. 設中斷點，程式跑到這裡會停下來
  3. 開始跑程式(run)，讓程式停在中斷點上
  4. 慢慢追程式
  5. 檢查一些變數值

載入程式

-g 可加入除錯資訊

gcc -g xxx.c -o xxx

這些除錯訊息會使檔案大小差10倍之多，所以一般在發佈應用程式時是不會以-g參數編譯的
可以在事後利用 strip 指令清掉應用程式裡的除錯資訊

strip ./xxx

開始除錯(不要做 optimize，不然程式碼行數會對不上)

gdb ./xxx
gdb -d /home/test ./xxx ##必要時加入-d (dirctory)，指定 source code 的位置
gdb參數 簡寫 說明
-symbols file_name
 -s 讀取檔案中的除錯表
-exec file_name -e 除錯一個執行檔
-se file_name 上二者的縮寫
-core file_name -c 讀入一個core dump檔案(系統當機之後用)
-pid process_id -p 啟動attach模式，除錯一個執行中的行程。number是目標行程的pid
-directory directory_name -d 將directory加入原始碼的搜尋路行
-readnow -r 一次讀取完所有的符號表，這會讓啟動gdb的時間
變長，但在執行往後的除錯動作會較快速。
-quiet/-silent -q 安靜模式，啟動時gdb將不會顯示版權頁。
-nowindows -nw 如果gdb有編入GUI的話，這個選項會關掉它
-windows -w 與上一選項相反，這會啟動GUI
-cd directory_name 切換工作目錄為directory而不是現在的目錄
-tty device_name -t 指定device 為程式的標準輸出入
—args 這個參數要當作命令列的最後一個參數，其後跟隨的參數
都會被視為「欲傳給將除錯的程式的參數」

設中斷點

參數 停止時機
break [function] 某函數的進入點
break offset 在程式停止位置的前(offset為負)/後第offset行
break linenum 指定行號
break filename:linenum 在file的指定行號或指定函式(若多型，需額外提供參數)
break 在下一個要執行的指令
break [args] if [cond] 當[cond]這個運算式為真，設定中斷點
args可能是上列的任一種情形
rbreak [RE] 符合正規運算式的函式
EX:(gdb) rbreak . 這樣每個函式開頭都有中斷點了
tbreak [args] 只會生效一次
指令 說明
info break(i b) 列出目前所有的中斷點
condition 進一步設中斷點的條件 如果條件為true則中斷
commands 在遇到中斷點時要自動執行的指令
clear 刪除所在行的中斷點
delete(d) breakpoint_id 刪除指定中斷點

開始跑程式

指令 說明
file 開啟檔案 (等同於 gdb filename)
run(r) 執行程式 (可加餵給程式的命令列參數)
kill 終止指令的執行
set 設定特定參數(或變數)。如：set env，設定環境變數。
unset 取消特定參數。如：unset env，刪除環境變數。
show 顯示特定參數。如：show environment，顯示環境變數。
set/show args 設定/顯示 命令列的參數
attach PID 載入正在執行的程式以進行除錯。PID 可由 ps 指令取得
detach PID 釋放已 attach 的程式

慢慢追程式(step, next …)

指令 說明
info line 查看正停在哪一行
frame 顯示正在執行的行數、副程式、及其所傳送的參數
where 顯示目前所在的frame的位置
backtrace (bt) 堆疊追蹤。會顯示出上層所有的 frame 的簡略資訊
next(n) 直接跑完整個frame，不會進入frame中
i.e. 每次執行主程式的一個指令
step(s) 不會直接跑完，而是進入frame中一步一步執行(※註)
i.e. 每次執行程式中的一個指令
until 直接跑完一個 while 迴圈
return 直接中止執行該 frame（視同該 frame 已執行完畢），並返回上個 frame 的呼叫點
等同C語言指令return;
finish 執行完目前的frame。
可能必須下達多個 finish 才能回到原來的進入點
up 回到上一層frame，frame被呼叫的地方，並顯示其 stack 資訊
Ex. up 2: 回到上三層frame
down 到下一層frame

※註：若該函式的除錯資訊沒有編進執行檔裡的話，那step也不會跳進這個函式裡，而是單純的將它看作一行程式碼(如同next的作用)，如標準函式庫(如stdio.h)提供的函式。

檢查變數值

print(p)

輸入p/?，?是格式字元，與printf不同的以粗體表示

x 格式字元
d 整數
u 無號整數(unsigned)
o 八進位
t 二進位
a 位址
c 字元
f 浮點數
指令 說明
whatis 印出變數的型態
p arr[1]@5 印出陣列[1]~[5]
存取臨時變數
(gdb) p i
$1 = 6078

$1 是執行 print 指令後，gdb 產生的一個臨時變數
可以直接利用 $1 來取用這個變數

用於 print 及 disp 的參數名稱

$: 前一次的運算式    
$$: 兩次前的運算式
$7: 第七個運算式
$$7: 倒數第七個運算式
display

display在中斷點時會自動顯示變數值

相關指令 說明
info locals 一次列出所有區域變數的值
frame frame_num 切換 frame(印出區域變數時使用)

顯示資訊

可以顯示某些資訊以利於debug

列出原始碼
指令 說明
list(第一次) 列出現在執行的位置上下5行
list(持續輸入) 繼續印出之後的程式碼(類似page down)
list - 印出上一次list的程式碼的前一段程式碼(類似page up)
list a,b 印出第 a ~ b 行
list filename:number 列出某檔案的第幾行，檔案名可省略
list [function] 列出某函數的程式碼
show listsize 顯示現在一次印出幾行
set listsize 設定一次印出幾行
disable 不加參數將會暫時關閉所有設定值
enable 再次啟用
info: 檢視 breakpoint 和 commands 和 display
指令 說明
info frame 顯示更多的副程式資訊
info args 顯示傳給副程式的參數值
info locals 顯示該副程式內所有區域變數的值
info reg 顯示暫存器的值
info all-reg 顯示暫存器的值，包括數學運算暫存器
info handle 列出目前 signal 設定的狀態
info share 顯示共享函式庫資訊
信號管理

預設為 stop print noignore，也就是遇到 signal 時，GDB 會先攔截，並中斷程式
必要時可以改為 nostop noprint，讓程式本身去處理 signal
若下達 ignore 則是讓程式忽略此 signal

handle (signal) (operation)
thread管理
指令 說明
thread 查看目前在哪個 thread
thread 3 切換至 第三個 thread
thread apply all bt / thread apply all bt full 列出所有 thread 的 frame stack

其他指令(括號內為簡寫)

  • quit 結束 (Ctrl + C)
  • help(h) 求助 (可加指令名稱)
  • shell 執行 Shell 指令
